These are my current ideas on what to do with libavg. To make this thing fly and set prioritites, I'm depending on lots of input: u.zadow at gmx.de. Of course, I can't promise anything and I welcome help of any kind.
  • Support for hardware acceleration using DirectFB: Currently, when DirectFB is used as output layer, all rendering is done in software. The software renderer is pretty well optimized, and this is ok for older systems, but for systems that support it, the graphic card's processing power stays unused.
  • Support for dynamic creation of avg nodes.
  • Support for loading images from the network.
  • Stylesheets are nice in html. They'd be nice for avg, too.
  • Memory management: Currently, all images are loaded on startup. That's good for small- to medium-sized projects, but some kind of caching/preload scheme would be nessesary for larger projects.
  • Better Mac OS X support. Of course, Windows might happen, too.
  • Vector graphics support (svg?).
  • Support for realtime image processing operators (Brightness, Contrast, Gamma, Blur,...) and blend modes (Multiply, Screen, Soft Light, Hard Light,...) using the hardware fragment shaders available on modern graphics cards. Darken, Lighten and Additive Blending already work, even on older graphics cards.

System State

We've had libavg running on Debian, Gentoo and SuSE boxes, and it shouldn't be impossible to get running on a sane Linux system. I've also gotten it to work on Mac OS X, but that's broken at the moment.

Prerequisites

libavg uses a lot of third-party libraries. What follows is a list of packages you'll need. The package names are the names of the gentoo packages needed. For other distributions, the list should enable you to find what's needed. At the moment, libavg doesn't check during configure that these libraries are installed, so you'll probably get compiler errors if they aren't.

  • imagemagick (Tested with 6.2.2.3)
  • ffmpeg 0.4.9pre1 or later. 0.4.8 doesn't work. Actually, ffmpeg isn't needed directly, but it's base libraries libavcodec and libavformat are. Debian users need to add deb ftp://ftp.nerim.net/debian-marillat/ unstable main to /etc/apt/sources.list to get this package.
  • libsdl. Tested with 1.2.5 and 1.2.8 (only needed for OpenGL and non-fullscreen DirectFB support).
  • Working OpenGL in some form or other or
  • DirectFB (Tested with 0.9.22, 0.9.19 and up should work.)
  • libxml2 (Tested with 2.6.19.)
  • libfreetype (Tested with 2.1.9)
  • pango (Tested with 1.8.1)
  • Optional: libdc1394 for Firewire digital camera support. (Tested with 0.9.3)
  • python 2.4.x. 2.3.x doesn't work.
  • boost (Tested with 1.31.0).

avg itself

Download the tarball: libavg-0.2.0.tar.gz.

Unpack it and run ./configure, make, make install

or

do a cvs checkout: 

cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/avg login

When prompted for a password for anonymous, simply press the Enter key.

cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/avg co avg

Then ./bootstrap to generate configure.

Now run ./configure, make, make install as usual.

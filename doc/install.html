Hic sunt leones

Be warned that this is stuff under heavy development. libavg is not polished enough to be easy to install yet. We've had it running on Debian unstable and SuSE boxes, and it shouldn't be impossible to get running on a sane Linux system, but it isn't packaged well yet, so things will hiccup and stutter and spit out compiler errors and in general behave badly until tamed. Of course, at this stage I'm more than happy to help anyone trying to install the beast: uzadow at gmx.de.

Prerequisites

libavg uses a lot of third-party libraries. What follows is a list of packages you'll need. I am not sure that the package names are exact, but the list should enable you to find what's needed.

  • The image format libraries libjpeg62-dev, libpng-dev, libtiff-dev (all optional but highly recommended)
  • libavcodec-dev
  • libsdl1.2-dev (only needed for non-fullscreen support)
  • DirectFB. (from www.directfb.org. I've used ver. 0.9.19 and several cvs versions of this library. The debian developer package works as well.)
  • OpenGL (installed OpenGL headers are nessesary to compile at the moment, but libavg will run without OpenGL support if DirectFB is working.)
  • libxml2-dev
  • paintlib-2.5.devel (from www.paintlib.de. Make sure the library finds DirectFB and the image format libraries during configure.)
  • mozilla-dev (tested with versions 1.4 and 1.6)

libavg itself

Do a cvs checkout: 

cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/libavg login

When prompted for a password for anonymous, simply press the Enter key.

cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/libavg co libavg

Then ./bootstrap to generate configure.

Now run ./configure, make, make install as usual.

Do a cvs checkout: 

cvs -d:ext:uzadow@cvs.libavg.sourceforge.net:/cvsroot/libavg co libavg

Then ./configure, make, make install as usual.

Linux kernel config

For DirectFB support, you need a working framebuffer. Also, installing the low-latency and preemptive patches is recommended to get a stable framerate.

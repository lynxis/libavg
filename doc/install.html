Hic sunt leones

Be warned that this is stuff under heavy development. libavg is not polished enough to be easy to install yet. We've had it running on a Debian unstable and a SuSE box, and it shouldn't be impossible to get running on a sane Linux system, but it isn't packaged well yet, so things will hiccup and stutter and spit out compiler errors and in general behave badly until tamed. Of course, at this stage I'm more than happy to help anyone trying to install the beast: uzadow at gmx.de.

Prerequisites

libavg uses a lot of third-party libraries. What follows is a list of packages you'll need. I have had neither the time to figure out the exact dependencies nor am I sure that the package names are exact, but the list should enable you to find what's needed.

  • pkgconfig (for DirectFB)
  • libpango1.0-dev
  • The image format libraries libjpeg62-devel, libpng-devel, libtiff-devel (all optional but highly recommended)
  • slang-dev
  • nasm
  • libmpeg3-dev
  • libsdl1.2-dev (only needed for non-fullscreen support)
  • DirectFB. (from www.directfb.org. I've used ver. 0.9.19 and several cvs versions of this library without problems.)
  • libxml2-dev
  • libcurl2-dev (optional, for paintlib url image loading support)
  • paintlib-2.5.devel (from www.paintlib.de. Make sure the library finds DirectFB and the image format libraries during configure.)
  • gtk-devel
  • orbit-devel
  • mozilla-devel (we're using ver. 1.4)

libavg itself

Do a cvs checkout: 

cvs -d:ext:uzadow@cvs.libavg.sourceforge.net:/cvsroot/libavg co libavg

Then ./configure, make, make install as usual.

Linux kernel config

You need a working framebuffer. Also, installing the low-latency and preemptive patches is recommended to get a stable framerate.
